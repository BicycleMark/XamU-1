XAM290 Master-detail and Drawer Navigation in Xamarin.Forms

Exercise 1: Control the Master behavior of a MasterDetailPage

The goal of this exercise is to influence how a MasterDetailPage presents the master UI for drawer and master-detail UI and understand how the MasterDetailPage behavior changes on different device idioms.

To complete the exercise, you will need Visual Studio for Windows or macOS with the Xamarin development tools installed. You will also need either an emulator/simulator or a device to run the exercise on. Please see the setup page if you need help installing the Xamarin development environment.

Open the starter solution

In Visual Studio, open the Exercise 1 Start solution included with the downloaded or cloned course materials:

  1. Launch Visual Studio
  2. Open Exercise 1/Start/PopoverVersusSplit.sln

Explore the code

This is a minimal application to demonstrate master-detail and drawer navigation in Xamarin.Forms. All of the code you'll be using is in App.xaml.cs, and the UI is defined programmatically.

  1. Open App.xaml.cs.
  2. Find the constructor: notice it's defining both the data and the UI. This application allows you to switch between details for three (3) different colors.
  3. Find the assignment for masterDetailPage.MasterBehavior: notice it's set to MasterBehavior.Popover.
  4. Run the application on any platform/device. Use the menu icon in the top left to open the drawer and update the presented color details page.

Change the Master behavior

  1. Change the MasterBehavior assignment to Split and run on a phone device or simulator. Verify that it still user popover behavior.
  2. Change the MasterBehavior assignment to SplitOnLandscape and SplitOnPortrait and run the application on a tablet or desktop each time. Does it behave as expected?
  3. Run the application on a desktop. Does the behavior change if you resize the window? Do you see the bahavior you expect?

Exercise summary

In this exercise, you influenced the presentation of the master page when using a MasterDetailPage. The default bahavior may not be exactly as you expected and understanding how Xamarin.Forms changes behavior based on idiom and orientation is very important when creating applications that run on a large range of device types.

Note: Since we were just altering the MasterBehavior, there isn't any Completed solution for this exercise.
